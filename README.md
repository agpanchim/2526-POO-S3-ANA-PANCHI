Comparación entre Programación Tradicional y Programación Orientada a Objetos en Python

INTRODUCCIÓN:

En el presente trabajo se realiza una comparación entre la Programación Tradicional y la Programación Orientada a Objetos (POO), aplicadas al desarrollo de un programa en Python que calcula el promedio semanal del clima.  
El objetivo es identificar las diferencias en la estructura, organización y mantenimiento del código utilizando ambos paradigmas de programación.

Programación Tradicional
La Programación Tradicional se basa en el uso de funciones y una estructura secuencial del programa.  
En este enfoque, el problema se resuelve mediante funciones independientes para el ingreso de datos y el cálculo del promedio semanal.

Característcas:
- Uso de funciones.
- Código más directo y sencillo.
- Adecuado para programas pequeños.
- Menor organización cuando el programa crece.

Programación Orientada a Objetos (POO)
La Programación Orientada a Objetos organiza el programa en clases y objetos.  
En este trabajo se utilizó una clase llamada `Temperatura_POO`, la cual encapsula los datos y métodos necesarios para manejar la información del clima.

Características:
- Uso de clases y objetos.
- Aplicación del encapsulamiento.
- Código más organizado y reutilizable.
- Facilita el mantenimiento y la escalabilidad.

Comparación entre entre Programación Tradicional y POO

La Programación Tradicional se basa en el uso de funciones independientes para cada tarea, lo que hace que el código sea más simple y directo. 
Sin embargo, su reutilización es limitada, y a medida que el programa crece, resulta más difícil de mantener y organizar.
En cambio, la Programación Orientada a Objetos (POO) utiliza clases y objetos para estructurar el programa. 
Esto permite encapsular los datos y métodos relacionados, facilitando la reutilización del código y su mantenimiento. 
Además, el enfoque orientado a objetos promueve una mayor organización y escalabilidad, siendo más adecuado para proyectos de mayor complejidad.

CONCLUSIÓN
La Programación Tradicional es adecuada para problemas pequeños y sencillos, ya que permite una implementación rápida y directa.  
Por otro lado, la Programación Orientada a Objetos ofrece una mejor organización del código, facilitando su mantenimiento y reutilización, siendo más apropiada para proyectos de mayor complejidad.
